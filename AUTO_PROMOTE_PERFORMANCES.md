# Авто-перевод спектаклей «Афиша → Спектакли» и раздача ролей актёрам

## Идея

- Спектакль создаётся в разделе **«Афиша»** (`Perfomances.afisha = True`).
- Админ заранее задаёт **расписание показов** (несколько дат/времён) и **состав**
  (актёр + роль). Состав и роли **не видны обычным пользователям**, пока спектакль
  находится в «Афише».
- Когда **прошёл последний показ**, спектакль автоматически:
  1. переезжает в раздел **«Спектакли»** (`afisha = False`);
  2. его состав/роли становятся видны на странице спектакля;
  3. каждому актёру на его страницу добавляется пара (спектакль, роль).

## Модель данных

- `PerformanceShow` — показы спектакля: `performance` (FK), `show_datetime` (tz-aware),
  необязательный `ticket_url`.
- `PerformanceCast` — состав: `performance` (FK), `actor` (FK), `role`;
  пара (`performance`, `actor`, `role`) уникальна.
- `Perfomances.roles_propagated` — флаг, что роли уже однократно разданы актёрам.

Роли актёра по-прежнему хранятся в массивах `Actors.perfomances` /
`Actors.role_in_perfomances` и **остаются доступны для ручного редактирования** в админке.
Авто-раздача дописывает пары идемпотентно (без дублей).

## Логика перехода

Перевод выполняет функция `my_app1.models.promote_performance()` —
атомарно (`select_for_update`), идемпотентно, безопасно при параллельных вызовах.
Вызывается:

- периодической командой (см. ниже) — основной автоматический путь;
- из админки в `PerfomanceDetail.put`, если админ вручную переключил спектакль в
  «Спектакли».

## Запуск по расписанию (cron)

Перевод выполняется management-командой:

```bash
cd backend/app
python manage.py promote_past_performances           # выполнить перевод
python manage.py promote_past_performances --dry-run  # только показать, ничего не меняя
```

Команду нужно запускать периодически. Рекомендуется **системный cron** (или
systemd-timer / отдельный контейнер с supercronic) — это надёжнее, чем планировщик
внутри процесса, и не требует брокера.

Пример строки cron (каждую минуту), с виртуальным окружением проекта:

```cron
* * * * * cd /path/to/Antrakt/backend/app && /path/to/Antrakt/backend/venv/bin/python manage.py promote_past_performances >> /var/log/antrakt/promote.log 2>&1
```

Команду безопасно запускать сколь угодно часто: уже переведённые спектакли
повторно не обрабатываются, роли актёрам не дублируются.

## Таймзона

`TIME_ZONE = 'Asia/Krasnoyarsk'` (UTC+7) — театр находится в Норильске.
Время показа, введённое в админке (`datetime-local`), трактуется как местное
время театра. «Показ прошёл» определяется сравнением с `timezone.now()`.

## Видимость состава

- `GET /afisha/` — состав/роли НЕ отдаются; отдаётся расписание показов.
- `GET /perfomances/` (раздел «Спектакли», `afisha=False`) — состав/роли отдаются.
- `GET /perfomance<id>/` — для спектакля в «Афише» состав скрыт для обычных
  пользователей и виден только суперпользователю (для редактирования в админке).
